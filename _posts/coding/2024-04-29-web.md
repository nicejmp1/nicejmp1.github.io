# 모던 웹 개발의 패러다임 변화
최근 웹 개발은 과거와 비교했을 때 상당한 변화를 겪었습니다. <br>
개발 환경의 진화와 새로운 기술의 도입으로 더욱 효율적이고 사용자 친화적인 웹 개발이 가능해졌습니다. <br>
이러한 변화의 중심에 있는 몇 가지 트렌드와 기술을 소개해보겠습니다. <br>

## 모노레포(Monorepo) 구조
>모노레포는 여러 프로젝트를 단일 저장소에서 관리하는 구조를 의미합니다. 이를 통해 공유되는 모듈을 한 곳에서 관리할 수 있고, 코드의 재사용성과 협업 효율성이 높아집니다.

<장점>
- 코드 공유와 재사용이 편리하다.
- CI/CD 파이프라인을 간소화할 수 있다.
- 여러 프로젝트 간의 버전 관리가 용이하다.
<br>

<단점>
- 저장소의 크기가 커질 수 있다.
- 적절한 툴링이 필요하다.

<도구 추천>
- Lerna
- Nx
- Rush

# 프레임워크와 라이브러리의 발전

#### React Server Components
>React Server Components는 React의 새로운 렌더링 패러다임으로, 서버와 클라이언트 간에 컴포넌트를 분리하여 렌더링합니다. 이를 통해 초기 로딩 시간을 줄이고, 서버 측에서 데이터와 컴포넌트를 결합하여 클라이언트에 전달하는 방식으로 성능 최적화를 할 수 있습니다.

<특징>
- 서버에서만 실행되는 컴포넌트는 번들에 포함되지 않아 크기를 줄인다.
- 상태 및 이벤트 처리 로직을 간단히 한다.

````javascript
// 서버 컴포넌트
export default function Greeting({ name }) {
  return <h1>Hello, {name}</h1>;
}

````

#### Vue 3와 Composition API
>Vue 3는 향상된 성능과 새로운 기능을 갖춘 Vue.js의 최신 버전입니다. Composition API를 도입하여 기존 Options API보다 모듈화와 코드 재사용성이 개선되었습니다.

<특징>
- 반응성 시스템 개선
- 더 작은 번들 사이즈
- 컴포지션 API를 통한 코드 구조화
<br> 

````javascript
import { ref, onMounted } from 'vue';

export default {
  setup() {
    const message = ref('Hello, Vue 3!');
    
    onMounted(() => {
      console.log('Component mounted!');
    });
    
    return { message };
  }
}
````

# 웹 성능 최적화

#### 이미지 최적화
>이미지는 웹 페이지 로딩 속도에 큰 영향을 미치는 요소 중 하나입니다. 적절한 포맷, 크기, 압축을 사용하면 성능을 크게 향상시킬 수 있습니다.

<포맷>
- WebP: JPEG 대비 30% 더 작은 파일 크기
- AVIF: WebP 대비 더 높은 압축률

<크기 및 해상도>
- srcset과 sizes 속성 활용
- picture 요소를 사용하여 브라우저 별 포맷 제공


````html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Example image">
</picture>

````

웹 애플리케이션의 번들 크기를 줄이기 위해 코드 분할과 지연 로딩을 활용할 수 있습니다. 이를 통해 사용자가 필요한 시점에만 코드를 불러와 성능을 최적화합니다.

````javascript
import React, { Suspense, lazy } from 'react';

const LazyComponent = lazy(() => import('./LazyComponent'));

function App() {
  return (
    <div>
      <Suspense fallback={<div>Loading...</div>}>
        <LazyComponent />
      </Suspense>
    </div>
  );
}
````

# 백엔드 기술의 발전

#### GraphQL
> GraphQL은 Facebook에서 개발한 쿼리 언어로, 클라이언트가 원하는 데이터만 요청할 수 있는 기능을 제공합니다. 이를 통해 오버페칭(Over-fetching)과 언더페칭(Under-fetching) 문제를 해결할 수 있습니다.

<장점>
- 필요한 데이터만 선택적으로 가져올 수 있다.
- 단일 요청으로 여러 자원에 접근할 수 있다.

<단점>
- 설정과 학습 곡선이 높다.
- 캐싱 및 보안에 대한 추가 고려가 필요하다.

````graphql
query {
  user(id: "1") {
    name
    age
    posts {
      title
      comments {
        text
      }
    }
  }
}
````

#### Serverless Architecture
> 서버리스 아키텍처는 백엔드 인프라를 관리하지 않고도 서버리스 컴퓨팅 플랫폼을 활용하여 코드를 실행할 수 있는 환경을 제공합니다.

<장점>
- 인프라 관리 부담 감소
- 유연한 확장성 및 비용 절감
<br>

<단점>
- 콜드 스타트로 인한 지연 시간
- 플랫폼에 대한 의존성 증가
<br>

<플랫폼>
- AWS Lambda
- Google Cloud Functions
- Azure Functions

# 개발 워크플로우 개선
#### CI/CD 파이프라인
> 지속적 통합/배포(CI/CD)는 코드 변경을 자동으로 테스트하고 배포하여 개발 주기를 가속화하는 방법입니다. 다음은 주요 도구와 개념입니다.
<br>

- Jenkins: 오픈소스 자동화 서버로 다양한 플러그인을 통한 확장성이 뛰어남
- GitHub Actions: GitHub에서 제공하는 CI/CD 워크플로우
- GitLab CI: GitLab의 내장 CI/CD 기능

````yaml 
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '14'
      - run: npm install
      - run: npm test
````
